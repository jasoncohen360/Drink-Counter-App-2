# Last Call — Stage 7: Chat reactions + QR code

This update adds emoji reactions to chat messages and a scannable QR code on the Invite screen.

- **QR code** — no setup needed. It just appears on the Invite popup ("Show QR code").
- **Reactions** — need one small table.

**Time: ~2 minutes.**

---

## The one step (for reactions)

Supabase → **SQL Editor** → **New query** → paste → **Run**:

```sql
create table if not exists reactions (
  id         uuid primary key default gen_random_uuid(),
  event_id   uuid references events(id) on delete cascade,
  chat_id    uuid references chat(id) on delete cascade,
  person_id  uuid references people(id) on delete set null,
  emoji      text not null,
  created_at timestamptz not null default now()
);

alter table reactions enable row level security;
create policy "open reactions" on reactions for all to anon, authenticated using (true) with check (true);

alter publication supabase_realtime add table reactions;
```

The last line makes reactions show up live on everyone's phones. "Success" = done.

---

## How reactions work

- Tap the **+** under any message to pick: ❤️ 👍 👎 🥂 ❗
- Or **double-tap a message** to quickly ❤️ it.
- Tap a reaction again to remove yours. Counts show how many people used each.

## About the QR code

The QR appears on the Invite popup when you tap "Show QR code." It encodes your join link, so anyone can scan it with their phone camera to jump straight in. It stays the same all night — screenshot it and stick it on a table if you want.

**Honest note:** the QR image is generated by a free third-party service (api.qrserver.com), which means two things — it needs an internet connection to display, and your join link is sent to that service to make the image. For a party join code that's harmless, but I want you to know it's not generated entirely inside your own app. If you'd ever prefer it fully self-contained, that's a small future change (bundling a QR generator into the app).
